Demo 1

Satellite Location to Location OpenShift Service Access

F5 Distributed Cloud provides secured connectivity for services running within OpenShift clustered deployed in your IBM Satellite Locations. It does this without the need to expose your container services outside the cluster. This removes the need to secure service access between clusters with infrastructure firewalls and application security devices.

F5 Distributed Cloud includes SaaS managed network and application security policies including full WAF and advanced web BOT protection. These policies can be applied wherever locations you choose to publish your services.

Let's attempt to make a service call from within our {{ location }} cluster to:

http://remoteservice.default/demoservice

This service won't exist in our cluster until it is published in this location's distributed cloud site by the service name 'remoteservice'.

Demo 2

Satellite Location OpenShift Service
to Satellite Location ICD (IBM cloud databases)

F5 Distributed Cloud provides secured connectivity for services running within Location OpenShift clusters to other services created in IBM Satellite Locations.

F5 Distributed Cloud SaaS decouples the responsibilies of a database administrator deploying and auditing with IBM Cloud Databases and providing access the database from other Locations.

Let's attempt to connect to a Satellite location deployed database from within our {{ location }} cluster to:

postgress://remotedb.default:5432/appdb

The service for our database won't exist in our cluster until it is published in this location's distributed cloud site by the service name 'remotedb'.

Demo 3

Satellite Location to Poly-Cloud Hosted Services

F5 Distributed Cloud provides secured connectivity for services running in any VPC or private datacenter hosting a distributed cloud site cluster.

Distributed cloud sites can be deloyed within IBM Satellite OpenShift clusters, IBM cloud VPC, AWS, Azure, GCP, VMware, and KVM based IaaS platforms.

Distributed cloud sites can be deployed as VMs, on bare metal servers, and supported F5 hardware gateways. They extend your reach into branch offices, manufacturing floors, or managed service facilities.

Let's attempt to make a service call from within our {{ location }} cluster to:

http://azureservice.default/demoservice

This service won't exist in our cluster until it is published in this location's distributed cloud site by the service name azureservice. The actual service is securely routed through a Azure virtual host based distributed cloud site cluster.

Call Remote Azure Service

Demo 4

Internet to Satellite Locations or Distributed Sites

F5 Distributed Cloud provides a global any cast enabled backbone which can host services at the Internet edge with delegated DNS domains.

Every distributed cloud service is instrumented to provide application analytics and visibility through the SaaS console.

The distributed cloud Internet edges include the ability to deploy network DDoS mitigations and HTTP application level protections like WAF, advance BOT protections, and ML based policy alerting.
